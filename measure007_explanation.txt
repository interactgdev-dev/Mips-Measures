Measure 007 (CAD: Beta-Blocker Therapy - Prior MI or LVEF <= 40%) - Implementation Notes

Source reference:
- 2026_Measure_007_MIPSCQM.pdf

What this code now does (denominator-focused):
1) Age check:
   - Includes only patients age >= 18.

2) CAD/proxy check:
   - Accepts either:
     a) CAD diagnosis ICD list match
     OR
     b) Cardiac surgery proxy CPT list match
   - This is treated as CAD-or-proxy condition.

3) Encounter check:
   - Requires denominator encounter CPT list match.

4) Prior MI check for criteria 2:
   - Uses MI ICD list and marks prior MI path when ICD is present.

5) Denominator exclusions:
   - No explicit denominator exclusion codes are currently configured for this measure.
   - Structure exists in code to add exclusions later if needed.

6) Denominator eligibility:
   - Criteria 1 eligible: CAD/proxy + encounter
   - Criteria 2 eligible: CAD/proxy + encounter + prior MI
   - Final denominator eligibility: criteria 1 OR criteria 2.
   - Output mapping:
     - ICD7C1, CPT7C11, CPT7C12, CPT7C13 represent criteria-1 denominator components
     - ICD7C21, ICD7C22, CPT7C21, CPT7C22, CPT7C23 represent criteria-2 denominator components
     - M007 = 1 if denominator eligible, else 0
     - E007 = 0 if eligible, else 1

Numerator handling:
- Numerator logic intentionally removed for this measure.
- These fields are intentionally neutralized:
  - N007_MET = 0
  - N007_EXCEPTION = 0
  - N007_NOT_MET = 0
  - QDC007 = ""

Important note vs PDF:
- PDF criteria 1 includes LVEF <= 40 requirement and both criteria include an additional denominator encounter requirement.
- Current implementation is denominator-focused but does not fully enforce those two detailed denominator constraints yet.
